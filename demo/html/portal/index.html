TopJUI前端框架

纯HTML调用功能组件，不用写JS代码的EasyUI，专注你的后端业务开发！

EasyUI是比较流行的前端框架，适用于企业信息系统或网站后台管理，随着IE所占市场份额越来越小，EasyUI的兼容性也越来越强，但在开发过程中，对于EasyUI稍微复杂的组件功能，大多需要通过手写JS代码实现，因此增加了开发难度，降低了开发效率。 我们在企业信息系统的开发过程中，经过近两年时间的研发，通过封装EasyUI组件功能，同时在EasyUI原有组件的基础上，扩展了常用的组件功能，整合了常用的第三方插件，形成具有特色的，通过纯HTML调用组件功能的TopJUI前端框架，目前已成功应用在多个企业信息管理系统中。

关于开发平台

通过十年的企业级信息系统开发经验积累总结与沉淀，我们根据客户的通用需求及软件开发中遇到的问题，开发了TopJUI前端框架， 在满足客户需求的同时，大大提高了前端开发的工作效率。我们根据TopJUI框架的特性， 整合后端JAVA开发框架（Spring + Spring MVC + Hibernate）及第三方JAVA组件， 研发了适用于企业管理的信息化系统开发平台“易网时代企业管理信息系统开发平台”（以下简称“开发平台”），通过此开发平台， 我们实现了企业管理信息系统的快速开发，目前已应用在多个企业的信息管理系统中，提供了在一个平台下支撑数十个子系统， 几百个子模块同时运行的企业级信息管理系统解决方案。

开发平台演示地址： http://demo.topjui.com/system/index/index ，用户名/密码：topjui / topjui

开发平台通用功能

多数据源：实现数据库分布式部署或集成已有信息系统的数据，实现系统整合

HQL查询：可通过HQL（Hibernate查询）实现查询或新增、更新、删除等操作

SQL查询：可通过SQL（JDBC Template查询）实现查询或新增、更新、删除等操作

视图查询：可通过调用视图查询数据，实现表与表之间复杂的关联查询功能

存储过程：可通过封装好的方法调用存储过程，实现数据库的强大数据处理功能

RESTful风格：在控制器中自由配置请求地址参数，实现URL地址的RESTful风格

数据导出：封装通用的数据导出Excel功能，可导出每个表格界面中过滤查询数据

报表导出：通过调用封装的导出接口，根据模板文件导出Word、Excel、Pdf报表

Json输出：统一Json格式数据输出，一套后端程序同时实现网页端、移动APP端的数据调用

数据过滤：每个表格界面，自带数据过滤功能，可同时过滤多个字段完成过滤查询

自定义查询：每个表格界面，通过自定义查询条件，实现可自由组合的数据查询

数据排序：可结合数据过滤或自定义组合查询，实现单个字段或多个字段组合排序

主子表关联：在子表中新增数据时，提示先选中主表数据，点击主表数据过滤子表数据

按需加载：只有点击子表所在的Tab页时才加载子表或子页面的信息，按需加载提高效率

查找带回：查询勾选某行数据，将与该数据行相关的信息回填到当前表单的其它输入框中

自动完成：类似百度的搜索功能，输入关键字后自动过滤显示相关的信息，如查找人员

多级联动：根据前一个下拉框选择的内容，动态关联下一个下拉框中的内容，如省市县关联

批量处理：实现表格数据的批量提交，如批量审批通过，批量修改部分字段信息等

可编辑表格：在可编辑表格中双击行实现编辑，通过ajax请求与后台同步提交修改

文件上传：整合百度的webuploader插件，同时具有flash及Html5上传效果，轻松实现附件上传

图形报表：整合百度的echarts插件，可实现如柱状图、拆线图、饼图等图形报表生成

文字编辑器：整合百度的Ueditor编辑器，实现富文本的编辑，文章发布等

开发平台模块功能

栏目管理：设置文章发布的栏目信息，开发企业门户网站时可调用相关栏目

文章管理：文章发布管理模块，所有文章类的信息可在此发布在不同的栏目中

知识文档：文档管理模块，通用的附件上传功能附件数据也保存在此模块中

组织架构：通过树形结构实现无限级的人员组织架构，包括设置各级负责人

用户信息：系统用户基本信息，如需其它字段信息，可通过扩展表新增

角色授权：基于RBAC的权限控制系统，可控制权限到所有页面、按钮及链接

会员管理：会员管理模块，可通过前台会员注册或后台手动录入会员信息

基础配置：可自定义增加调整系统配置信息，在系统任何界面都可实现调用

系统菜单：系统后台的所有菜单，均在此模块进行配置，通过授权后显示在左侧导航中

代码生成：通过代码生成器一键自动生成Controller、Service层的通用增删改查代码

指标体系：指标体系即数据字典信息，通过无限级分类，实现数据字典的信息调用

日志记录：记录所有用户操作信息，包括请求地址、传递参数，耗时、IP地址等信息

异常记录：记录系统运行过程中的异常报错信息，及时主动发现问题，修复系统BUG
